Code skeleton

loop/controller.js — what step 1 actually writes

// caps live in code, not in the agent's head
const CAP_ITERS = 4, CAP_TOKENS = budget, CAP_WALL_MS = 120_000

async function run(brief, rubric, run_id) {
  let best = null, iter = 0, feedback = null
  const t0 = clock()

  while (
    iter < CAP_ITERS &&
    tokens() < CAP_TOKENS &&
    clock() - t0 < CAP_WALL_MS
  ) {
    iter++

    // ① builder — STUB on day 1: return canned EDL
    const edl = await builder(clips, feedback)
    await emit(run_id, iter, 'BUILD')

    // ② validate — real from day 1 (deterministic)
    if (!validateEDL(edl, clips)) { feedback = 'invalid edl'; continue }

    // ③④ render+overlay — STUB: return canned file ref
    const render = await renderAndOverlay(edl)
    await emit(run_id, iter, 'RENDER', render.ref)

    // ⑤ verifier — STUB: canned grade (fail iter 1, pass iter 2)
    const grade = await verify(render, rubric)
    await emitGrade(run_id, iter, grade)

    // ⑥ best-so-far — unconditional
    if (!best || grade.total > best.grade.total) best = { render, grade }

    if (grade.pass) break   // every dim ≥ 7
    feedback = grade.feedback   // ↻ loop
  }

  await writeMemoryRule(best)
  await emit(run_id, iter, 'DONE', best.render.ref, best.grade)
  return { shipped: best, stoppedAt: iter, passed: best.grade.pass }
}
Why the while-gate, not /goal: the agent never decides when to stop. Three caps in one boolean guarantee the loop terminates; best guarantees it ships something. That pair is your #1 demo-killer mitigation (§0 Loop Termination) — and it's testable with stubs, no ffmpeg or model needed.

The boundary that makes it work: only ① Builder and ⑤ Verifier are agent() calls (blue). Everything else — the while-gate, validate, best-so-far, render orchestration, emit — is plain deterministic JS (ink). Step 1 proves that skeleton with the two agents stubbed. Steps 2→5 of the build order then swap each stub for the real module (EDL renderer → overlay → verifier → builder) without touching this control flow. The interface freezes here.
