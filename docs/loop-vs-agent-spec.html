AutoCut · architecture change · Option C — autonomy with a deterministic backstop

What changes: the loop stops being code, and becomes Claude

Today a deterministic JS while-loop drives the run and calls the model twice per iteration (build, grade). After this change, a single autonomous Claude session (Agent SDK, on Daytona) drives itself — calling tools, deciding when it's good enough, and looping on its own judgment. The harness no longer runs the loop; it fences it with hard caps and watches the stream to keep best-so-far. The Supabase plumbing in agent-server.ts is untouched.

The new brain — src/agent/session.ts

The while-gate is gone. query() drives; the caps fence it; the stream loop emits + tracks best-so-far. Compare to controller.ts — the deterministic responsibilities survive, but they move from driving the loop to fencing it.

// caps are the BACKSTOP now — Claude drives, the harness fences
const CAP_TURNS = 12, CAP_BUDGET_USD = 0.50, CAP_WALL_MS = 120_000

async function runAgentSession(input, tools, emit) {
  let best = null
  // tools = in-process Agent SDK tools wrapping our modules (STUB impls on day 1)
  const server = createSdkMcpServer({ name: 'autocut', tools }) // search_clips·build_edl·render·grade·publish

  const q = query({
    prompt: goalFrom(input),            // brief + rubric + consulted memory rules
    options: {
      model: 'claude-opus-4-8',
      mcpServers: { autocut: server },
      allowedTools: ['mcp__autocut__search_clips', '…', 'mcp__autocut__publish'],
      permissionMode: 'bypassPermissions',  // headless — no prompts
      maxTurns: CAP_TURNS,               // ← SDK-enforced cap #1
      maxBudgetUsd: CAP_BUDGET_USD,       // ← SDK-enforced cap #2
    },
  })
  const timer = setTimeout(() => q.close(), CAP_WALL_MS)  // ← wall-clock cap #3

  for await (const msg of q) {
    if (msg.type === 'assistant') emitToolSteps(msg, emit)        // ◆ events row per tool call
    if (isGradeResult(msg)) best = bestOf(best, parseGrade(msg)) // best-so-far, unconditional
    if (msg.type === 'result') {                              // success | error_max_turns | error_max_budget_usd
      clearTimeout(timer)
      return { shipped: best, passed: best?.grade.pass ?? false, stop: msg.subtype }
    }
  }
}
Same contract, swapped engine: runAgentSession returns the exact { shipped, passed } shape runLoop did — so executeRun's distill → upload → status tail, and all of agent-server's Supabase plumbing, never change. The swap is one call site.

The one idea to take away: the boundary between "model" and "deterministic code" doesn't disappear — it moves. Before, JS held the loop and the model filled in two blanks. After, the model holds the loop and JS draws a fence around it (caps) and a camera on it (stream-watch → events + best-so-far). The PRD's #1 rule — "never trust the model to self-terminate; always ship something" — is enforced exactly as before, just by maxTurns/maxBudgetUsd/close() instead of a while condition.
