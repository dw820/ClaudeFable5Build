The Preprocess Pipeline

Offline producer of clips.json — turns a folder of raw videos into a searchable, validated clip library.

Preprocess runs once, offline, before the editing agent ever wakes up. It walks a media directory and, for each clip in parallel, extracts three things — what's said (transcript), what's shown (caption + tags), and the container facts (duration + resolution). Those parts are assembled into one validated ClipLibrary, uploaded to Supabase Storage, and written to clips.json. Everything below uses a real example: a beach-trip project.

Discover the clips

cli.ts · runDirectory()

Read the media directory, keep only files with a video extension (.mp4 .mov .m4v .webm .mkv), and sort them for a stable order. Empty directory → hard error (nothing to do). Each file id is its basename, sanitized.

Input — directory listing readdir
media/
  wipeout.mp4
  sunset.mp4
  paddle.mp4
  notes.txt        ← skipped (not video)
  campfire.mov
  …
→filter + sort
Output — clip work-list string[]
[
  "wipeout.mp4",  // id: "wipeout"
  "sunset.mp4",   // id: "sunset"
  "paddle.mp4",
  "campfire.mov",
  …
]
Fan-out: every file below runs through Steps 1–3 concurrently via Promise.all. We deep-dive a single clip — wipeout.mp4 — then show the whole library at the end.
1

Transcribe — what is said

transcribe.ts · transcribeClip() → whisper on Replicate

The clip's audio is sent to OpenAI Whisper (via Replicate) asking for word-level timestamps. The raw response is normalized by the pure parseWhisperWords() into a flat list of { word, t0, t1 } tokens. A clip with no decodable audio (b-roll, silent screen-capture) degrades gracefully to [] — it does not fail the batch.

Input — request openai/whisper
{
  audio: <wipeout.mp4 bytes>,
  language: "auto",
  timestamp: "word",
  word_timestamps: true
}
→parseWhisperWords
Output — TranscriptWord[] {word,t0,t1}
[
  { word: "watch", t0: 0.2, t1: 0.5 },
  { word: "this",  t0: 0.5, t1: 0.8 }
]

// silent clip (e.g. sunset.mp4) → []
Why timestamps matter: downstream the agent uses t0/t1 to place burned-in captions in sync with speech. Whisper output shape varies by build (chunks / segments / words); the parser accepts all three.
2

Understand frames — what is shown

frameUnderstand.ts · understandFrames() → ffmpeg + LLaVA on Replicate

ffmpeg samples a few evenly-spaced JPEG frames; the middle (representative) frame is sent to a vision-language model (LLaVA-13B) prompted to return one vivid caption plus 3–6 short content tags as strict JSON. parseVlmResponse() tolerates loose prose too — a non-conforming model still yields a usable caption (tags empty).

Input — frames + prompt llava-13b
{
  image: "data:image/jpeg;base64,/9j…",
  prompt: "Describe this clip in one
    vivid sentence… then 3-6 tags.
    Respond as strict JSON."
}
→parseVlmResponse
Output — caption + tags {caption,tags}
{
  caption: "a surfer paddles hard and
     wipes out on a breaking wave",
  tags: ["action", "hook",
          "ocean", "wipeout"]
}
Tags are normalized — lowercased, trimmed, de-duped, capped at 8. The caption is also what gets embedded for semantic search in Step 5.
3

Probe — the container facts

cli.ts · probeClip() → ffprobe

ffprobe reads duration and the video stream's width/height. These become the clip's duration and resolution. A non-positive duration is treated as a probe failure and fails loud — a zero-length "clip" is never allowed into the library.

Input — ffprobe query ffprobe -of json
-show_entries
  stream=width,height
  :format=duration
wipeout.mp4
→parse JSON
Output — ClipMeta {durationS,w,h}
{
  durationS: 12,
  width:     1080,
  height:    1920
}
4

Assemble & validate — join the parts

assemble.ts · buildClip() → assembleLibrary() → ClipLibrarySchema.parse()

The three parts for each clip are merged into one Clip (the whole source clip is one selectable unit, so start = 0, end = duration). All clips combine into a ClipLibrary that is validated by a Zod schema — this is the gate. A malformed clip (zero dimension, out-of-spec transcript) is caught here, not three modules downstream in the editor.

Input — the 3 parts for one clip
transcript ┐
caption+tags ┼─ buildClip("wipeout")
meta ┘
→buildClip
Output — one validated Clip Clip
{
  id: "wipeout",
  src: "wipeout.mp4",
  start: 0, end: 12, duration: 12,
  resolution: [1080, 1920],
  transcript: [{ word:"watch",t0:0.2,t1:0.5 }, …],
  caption: "a surfer paddles hard…",
  tags: ["action","hook","ocean","wipeout"]
}
5

Upload to Storage

storage.ts · uploadLibraryAssets() → Supabase Storage (source-clips bucket)

When SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY are set, every clip's bytes are uploaded first, then the manifest is uploaded last — so a published clips.json can never reference a clip that failed to upload. Each clip's src is rewritten to a storage:// ref. (No Supabase env → pure-local dev, src stays a relative path.)

Upload order bucket: source-clips
// clips first…
beach-trip/wipeout.mp4   video/mp4
beach-trip/sunset.mp4    video/mp4
beach-trip/campfire.mov  video/quicktime
…
// …manifest LAST
beach-trip/clips.json    application/json
→src rewrite
Output — rewritten refs storage://
src: "wipeout.mp4"
   ↓
src: "storage://beach-trip/wipeout.mp4"

// the agent worker later downloads
// these with the service-role key
Asset model: Supabase Storage is the source of truth; the agent's Daytona disk is just a cache. Preprocess uploads, the worker downloads-and-caches on demand.
OFF the demo path — wired & tested, runs only when enabled
★

Embed & index for semantic search

embed.ts (all-mpnet-base-v2) → pgvector.ts → Supabase clip_embeddings table

Each clip's caption is embedded into a vector and upserted to a Postgres pgvector table, keyed on clip_id so re-running preprocess is idempotent. This powers later "find me clips like X" retrieval. It never touches the ClipLibrary contract, so it stays off the critical path.

Input — caption text embedText
{ text: "a surfer paddles hard and
          wipes out on a breaking wave" }
→upsert
Output — pgvector row clip_embeddings
{
  clip_id:    "wipeout",
  project_id: "beach-trip",
  caption:    "a surfer paddles hard…",
  embedding:  [0.013, -0.072, 0.041, …768]
}
